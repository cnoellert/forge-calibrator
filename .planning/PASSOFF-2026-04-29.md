## Per-task summary

### `260429-ebd` — 10/12-bit decode + dialog branch

`decode_raw_rgb_buffer` now handles bd=10 (DPX method A BE DWORD: `>>22 / >>12 / >>2`,
÷1023) and bd=12 (uint16 ÷65535, BRG perm). Previously only 8/16/32 worked; 10 and 12
returned None and surfaced as the misleading "media path inaccessible" dialog.

The hook now branches the dialog: real path failures stay on the original message;
decode rejection on a known bit_depth gets *"Calibrator preview doesn't support N-bit
clips yet"*. UAT'd on portofino across `testImage`, `testImage10bit`, `testImage12bit`,
`C002_260302_C005`, `A005C008_120101_NQ96` — all clean.

**Plan note:** bd=12 deviated from the plan's `÷4095` to `÷65535`. Bridge probe revealed
Wiretap left-shifts 12-bit values to fill the full 16-bit range; the plan's assumption
was wrong, the executor caught it via Rule 1.

### `260429-fk5` — Blender binary glob defaults

`_DEFAULT_BLENDER_BINS` and `resolve_blender_bin()` now glob-expand candidates and
expanduser-resolve `~/...` paths, descending-sort multi-matches. New defaults cover:

- linux: `/opt/blender/blender`, `/opt/blender-*/blender`, `~/Apps/blender-*/blender`
- darwin: `/Applications/Blender*.app/Contents/MacOS/Blender`,
  `~/Applications/Blender.app/...`, `~/Applications/Blender*.app/...`

Env override (`FORGE_BLENDER_BIN`) and PATH fallback unchanged. flame-01 currently has
Blender symlinked into `/usr/local/bin` for testing; once that symlink is removed the
new `/opt/blender-*/glob` should pick up wherever the real install is.

Backlog todo filed at `2026-04-29-blender-binary-picker-ui-tweak.md` for an in-UI
"Browse for Blender..." affordance — low priority, captured for power users on weird
layouts.
