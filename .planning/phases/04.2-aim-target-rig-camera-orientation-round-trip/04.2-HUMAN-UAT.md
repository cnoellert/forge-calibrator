  Blender→Flame round-trip is hard-crashing Flame. User-observed symptoms during 2026-04-25 live UAT:

    Python encountered ModuleNotFoundError("No module named 'bpy'") in unknown code
    [PYTHON HOOK] An error occurred. Ignoring python hooks from /opt/Autodesk/shared/python/blender/forge_sender/flame_math.py

  followed by a hard crash in Flame when "trying to pass back to flame".

  History: this test was briefly marked `pass` on 2026-04-25 based on Phase 04.3's
  unit-test math closing the 0.087° ry residual to 0.006°. That marking was
  premature — the unit-test math is correct, but the user-facing round-trip is
  failing for a separate reason exposed by live UAT.

  Likely root cause (unconfirmed): Phase 01-VERIFICATION.md Test 1's rsync
  command (`rsync -av flame/ forge_flame/ forge_core/ tools/ /opt/Autodesk/
  shared/python/`) deploys `tools/blender/` INTO Flame's hook discovery tree
  at `/opt/Autodesk/shared/python/blender/`. install.sh:45-50 deliberately
  routes `tools/blender/` to `/opt/Autodesk/shared/forge_blender_scripts/`
  OUTSIDE that tree because forge_sender/flame_math.py top-imports `bpy`
  which Flame's Python lacks. The UAT instruction sidesteps that guard.

  Whether the bpy hook-load error and the hard crash share a root cause is
  unconfirmed. Two hypotheses:
    H1: Same root cause — the misplaced module corrupts Flame's hook state,
        triggering instability that surfaces as a crash on import_fbx.
    H2: Independent — bpy hook error is hook-scanner noise; the crash is a
        Phase 04.2-class importToActionFBX failure (which Phase 04.3 was
        supposed to mitigate via atomic-symmetric landing).

  Triage steps recommended for user:
    1. ls /opt/Autodesk/shared/python/blender/  → confirm misplaced files exist
    2. rm -rf /opt/Autodesk/shared/python/blender/  → remove them
    3. ./install.sh  → redeploy correctly
    4. Restart Flame, retry the round-trip

  If after step 3 the hard crash persists, escalate to /gsd-debug with the
  Flame log captured at the crash time.
