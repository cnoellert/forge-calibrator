## Decisions Made

- **Roll sign convention (Rule-1 fix during TDD):** The initial implementation used `theta = np.radians(+roll_deg)` which produced `rz = +1.2521°` for Camera1 input roll `-1.252521°` (opposite sign). The D-07 test explicitly expects `|rz - roll_deg| < 1°`. Fix: negate theta, so `theta = np.radians(-roll_deg)`. Interpretation: Flame's Roll property is the clockwise roll as seen looking DOWN -forward (the lens-through view), which corresponds to rotating right/up_cam about +forward by -roll_deg. This sign is now locked by both the Camera1 case and the dedicated test_roll_sign_convention test.
- **Grep acceptance criterion interpretation:** The plan required `grep -nE "import cv2|from forge_flame|import flame" rotations.py` to return 0 matches. Actual result: 1 match, but it is a pre-existing docstring line (line 22) referencing `forge_flame.adapter`'s re-export pattern — not an import statement. AST inspection confirmed the module's only imports are `__future__`, `typing`, and `numpy`. The spirit of the criterion (module stays numpy-only per D-06) is fully met.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Roll sign in Rodrigues rotation was inverted**

- **Found during:** Task 2 (`test_camera1_known_answer` failed — `rz=+1.252°` for input `roll=-1.252°`, 2.5° error vs 1° tolerance).
- **Issue:** Initial implementation used `theta = np.radians(float(roll_deg))` — rotates right/up_cam about +forward by +roll. But Flame's Roll property rotates clockwise looking down -forward (through the lens), which is equivalent to rotating about +forward by -roll. The free-rig cross-check with `roll_deg=0` still passed because 0 has no sign, masking the error until the Camera1 live-probe case exercised non-zero roll.
- **Fix:** Changed `theta = np.radians(float(roll_deg))` to `theta = np.radians(-float(roll_deg))`. Added expanded inline comment citing the Camera1 live probe values and the semantic reason for the sign.
- **Files modified:** `forge_core/math/rotations.py` (Rodrigues block inside `rotation_matrix_from_look_at`).
- **Verification:** `test_camera1_known_answer` passes with `rz=-1.252091°` (4e-4° from input roll). `test_roll_sign_convention` still passes. All 4 `test_free_rig_cross_check` parametrize cases still pass at atol=1e-9. Full suite: 372 passed, 2 skipped.
- **Committed in:** `be18ce3` (fix commit, separate from feat/test commits for clean atomic history and traceability).
