**Implementation detail — `print(file=f)` vs `f.write(...\n)`:**

Inside `_FLAME_SIDE_TEMPLATE = '''...'''`, Python processes escape sequences when parsing the outer string. A `\\n` in source becomes `\n` (real newline) in the parsed string. When exec'd, this breaks a string literal like `f.write("...\n" % msg)` across lines with a `SyntaxError: unterminated string literal`. Fix: use `print("[forge-send] %s" % msg, file=f)` — print adds the newline without any escape sequence in the template source.

**Corollary:** Comments inside the triple-quoted template must not contain backslash sequences (`\n`, `\\n`) — they become real newlines in the parsed string and cause `IndentationError` when exec'd.
