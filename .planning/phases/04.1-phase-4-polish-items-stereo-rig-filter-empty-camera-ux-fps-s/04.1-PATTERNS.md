```python

stamped = cam.data.get("forge_bake_frame_rate")

    label = str(stamped)
```

**Invariant item 5 must preserve**: the stamped string MUST be one of `_FLAME_FPS_LABELS[0][0]` values (`"23.976 fps"`, `"24 fps"`, `"25 fps"`, `"29.97 fps"`, `"30 fps"`, `"48 fps"`, `"50 fps"`, `"59.94 fps"`, `"60 fps"`). **The Flame-project fps is NOT a label string today** — it's whatever `flame.batch.frame_rate` exposes. **Source of truth:** `forge_flame/fbx_io.py:63-70 iter_keyframable_cameras`

```python
    continue
```

**Why:** Unit-testable without importing `flame`; survives Flame API shape changes; matches the established precedent at `forge_flame/fbx_io.py:65` and `forge_sender/transport.py:113` (`hasattr(n, "import_fbx")`). CLAUDE.md § Duck Typing codifies this as the project standard.

### P-2: Defensive PyAttribute read with silent safe-fallback
**Source of truth:** `flame/camera_match_hook.py:2148-2160` (start_frame read)
**Apply to:** Any new `flame.*.get_value()` read item 2 or item 5 introduces
**Excerpt:**
```python
# Default values form a safe no-op (paired invariant)
var_a = <no-op default>
var_b = <no-op default>
try:
    raw = flame.X.Y.get_value()
    var_a = int(float(raw))       # defensive coercion handles int/float/str
    var_b = <derived from var_a>
except Exception:
    # Keep the defaults — NEVER let an unknown PyAttribute shape crash
    pass
```
**Why:** PyAttribute returns are not shape-guaranteed (`int` / `float` / `str` depending on attr type). The paired invariant rule — fallbacks must compose into a safe no-op — is what prevents silent data loss.

### P-3: Custom-property reserved-key guard
**Source of truth:** `tools/blender/bake_camera.py:145-202 _stamp_metadata`
**Apply to:** item 5's `forge_bake_frame_rate` addition
**Excerpt:** See Site 5 above.
**Why:** Caller-supplied `custom_properties` must never clobber round-trip-critical stamps. Pattern established by Phase 1 `01-REVIEW.md` WR-01 fix (committed already). Item 5's new stamp is round-trip-critical (the whole point of D-11 "always stamp"), so it joins the frozenset.
