### Auto-fixed Issues

**1. [Rule 1 - Bug] sys.modules stub pollution across test modules**

- **Found during:** Task 3 GREEN phase (full regression sweep)
- **Issue:** `test_hook_export_camera_to_blender.py` installed `cv2` MagicMock via `sys.modules.setdefault` which persisted into `test_image_buffer.py` (which needs real cv2). Similarly, `test_bake_camera.py`'s mathutils stub persisted into `test_forge_sender_flame_math.py`.
- **Fix:** Applied sys.modules restore pattern â€” record pre-stub state, install stubs, import module, restore. Stubs are removed after the hook/bake_camera module load so they don't pollute subsequent test collection.
- **Files modified:** `tests/test_hook_export_camera_to_blender.py`, `tests/test_bake_camera.py`

**2. [Rule 2 - Missing critical] `start_frame_int` default for static branch**

- **Found during:** Task 3 implementation review
- **Issue:** The original start_frame read block set `start_frame_int` inside the try block, leaving it undefined in the except path. The static branch needs `start_frame_int` as the `frame=` kwarg even when the read fails.
- **Fix:** Added `start_frame_int = 1` as a conservative default BEFORE the try block (paired with `frame_offset=0, frame_start=None` as the existing safe no-op).
- **Files modified:** `flame/camera_match_hook.py`
