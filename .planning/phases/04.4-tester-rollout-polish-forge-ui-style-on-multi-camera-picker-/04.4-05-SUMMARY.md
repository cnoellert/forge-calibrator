## Why

CONTEXT D-13 mandates a user-facing rebrand from "Camera Match" to "Camera Calibrator" while keeping internal Python module / install-path names unchanged (Camera Match → camera_match_hook.py filename stays). RESEARCH §R-08 enumerates every doc string requiring update. Plan 04.4-03 flattened the Flame batch menu to `FORGE → ...` (no Camera sub-tier) and Plan 04.4-04 updated the Blender addon's user-visible copy to match. This plan closes the loop on the only user-facing in-repo documentation file so testers reading `docs/seamless-bridge.md` see the same menu structure and brand the addon and hook actually emit.

The primary risk this plan averts: a tester triggering the bridge-unreachable error sees the addon say "Camera Calibrator hook loaded" (per Plan 04.4-04 at L386/L439/L508) but reads the doc's troubleshooting recipe headed by "Camera Match hook loaded" — they would assume the doc is for an older release and abandon the recipe. After this plan, doc symptom headings match the addon's report() output verbatim for the Plan 04.4-04 paths.
