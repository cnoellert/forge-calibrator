## Decisions Made

- **Flat FORGE group, no submenu retry.** RESEARCH §P-01 had already verified that Flame 2026.2.1 ignores nested `actions` arrays under `actions` arrays at the batch hook level — the inner array silently collapses. The plan body's "P-01 correction to D-12" note made this explicit; followed without re-probing.
- **Reworded the get_batch_custom_ui_actions docstring** so the literal substring `Import Camera from Blender` does not appear (the acceptance criterion demands `grep -c "Import Camera from Blender" = 0`). Substituted "The legacy 'Import' entry (Blender-to-Flame pull) was removed (D-06)…". Tracked as a Rule 1 wording fix below.
- **Reworded the residual comment** inside `_export_camera_pipeline` that mentioned `QFileDialog` for the same reason (`grep -c "QFileDialog" = 0`). Substituted "input/file-open dialogs" — generic phrasing, same explanation. Tracked as a Rule 1 wording fix below.
- **Replaced the Apply Camera QInputDialog with `_pick_camera` via sentinel injection.** The Apply Camera flow needed a "Create new Action" option that doesn't fit `_pick_camera`'s `(action, cam, label)` return shape. Solution: append `(None, None, "Create new Action")` to the cameras list before calling `_pick_camera`, then detect the sentinel by label after return. Preserves the existing two-path behavior (pick existing OR create new) inside a single FORGE-styled dialog. Rule 3 deviation per orchestrator scope_boundary.
- **Left line 243 `# PySide2 Camera Match Window` untouched.** Section-banner comment naming the `CameraMatchWindow` class — internal symbol per D-13 internal-name guard. Not user-rendered. The R-08 audit table doesn't list line 243.
- **Did not change `tools/blender/*` or `docs/*` or any non-hook file.** Plan 04.4-04 owns the addon; 04.4-05 owns the docs.
