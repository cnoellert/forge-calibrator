### IN-03: `_FakeObject.__init__` annotation could tighten

**File:** `tests/test_forge_sender_preflight.py:35`
**Issue:** Signature is `def __init__(self, type_: str = "CAMERA",
data: dict = None)`. The `dict = None` form is the correct Python idiom
(line 37 handles the None via `_FakeData(data or {})`), but the type
hint should strictly be `Optional[dict]`. With `from __future__ import
annotations` in effect (line 12) the annotation is a string at runtime,
so this is harmless and matches the project's prevailing test-fake
style (see `_Camera`, `_Action` in `tests/test_fbx_io.py`).
**Fix:** Optional. Tighten to `Optional[dict] = None` for strict
type-checker happiness. Not blocking — existing code passes pytest
cleanly.
