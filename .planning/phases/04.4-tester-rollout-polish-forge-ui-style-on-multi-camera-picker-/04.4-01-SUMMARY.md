## Decisions Made

- **Reused existing fake-flame harness** (importlib.util loader + sys.modules stub install + post-load restoration) from `test_hook_export_camera_to_blender.py` rather than inventing a new pattern. Keeps test conventions consistent across the hook test suite.
- **Skip-gate uses `hasattr(transport, ...) AND hasattr(transport, ...)`** so partial Wave-3 landings can't silently flip the gate (e.g. if `list_batch_actions` lands but `make_create_code` doesn't, the tests stay skipped).
- **Pitfall-1 canary self-asserts its premise** — the test sanity-checks `isinstance(item.type, str)` and `not hasattr(item.type, 'get_value')` BEFORE invoking the helper. Prevents the canary from silently rotting if a future test refactor changes the fake.
- **Source-text scan drives R-08 skip** instead of a manual flag — `pytest.skip(...) if "Camera Match" in src` automatically flips to PASS the moment Wave 3 finishes the rename, with no separate ledger to update.
- **Did NOT flip `VALIDATION.md wave_0_complete: false → true`** — per the plan's `<how-to-verify>` step 4, that flip is the human's responsibility at the Task 4 checkpoint. The agent's job is to make the verification trivial; the user's job is to acknowledge it.
