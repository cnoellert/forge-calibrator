## Decisions Made

- **Preserve helper signature.** The function name, parameter list, and `(action, cam) | (None, None)` return shape stay identical so the existing caller `_export_camera_from_action_selection` (line 2284) doesn't need changes. Trailing path returns `(None, item)` when cam is found but no Action contains it — caller checks `action_node is None` so behavior is identical to `(None, None)`.
- **Identity comparison in scan fallback.** `inode is item` rather than name equality. Name equality would be wrong when two Actions both hold a 'Camera1'; identity is the only reliable disambiguator.
- **Rename block inserted in template, not extracted to a helper.** The duck-type predicate mirrors the post-rename filter at lines 198-200 verbatim; keeping them adjacent makes any future template-text canary catch divergence immediately.
- **No addon zip rebuild.** The addon at `tools/blender/forge_sender/__init__.py:551` already stamps `forge_bake_camera_name` correctly; both legacy stamped path and the new `FORGE_OT_send_to_flame_choose_action` path are covered by the bridge-side template fix alone.
- **Empty-name no-op guard.** `if forge_bake_camera_name:` — Python treats empty string as falsy, so a stamped-but-empty value is a no-op preserving the FBX-default name. Matches HUMAN-UAT GAP-2 fix scope.
- **Test gating uses skip pattern with grep-able message.** `_first_camera_helper_has_fallback()` checks for the substring `"for n in flame.batch.nodes"` inside the function source, distinguishing Wave-1 deferred shape from Plan 04.4-07 rewrite.
