def _first_camera_in_action_selection(selection):
    """Return (action_node, cam_node) for the first non-Perspective Camera
    in selection, or (None, None) if no such item.

    cam_node.parent is the containing PyActionNode (verified live via
    forge-bridge probe 2026-04-25; see RESEARCH P-02). Same iterate-and-

def make_create_code(name: str) -> str:
    """Build the Flame-side Python that creates a new Action and renames it.

    The user-provided name is embedded via repr() — never f-stringed —
    per the transport.py security note (T-02-03-01). On Flame side,
    name.set_value() raises RuntimeError on collision (RESEARCH P-03,
    Pitfall 2). Caller surfaces that error to the user.

    Returns the Python code string ready to POST as {"code": <here>}.
    The bridge response's "result" field will be the new Action's name
    (repr() of a str — parse with ast.literal_eval).

    if not isinstance(name, str):
        raise TypeError(
            f"make_create_code: expected name to be str, got "
            f"{type(name).__name__}")

        "import flame\n"
        "_n = flame.batch.create_node('Action')\n"
        f"_n.name.set_value({name!r})\n"   # !r == repr() — escapes quotes

**Why:** `camera_match_hook.py` is imported at Flame startup. Top-level `import flame` would create a circular dependency. Every scope helper and handler in the file uses lazy imports.

### `_val(attr)` helper for PyAttribute reads

**Source:** Used throughout `flame/camera_match_hook.py` (e.g. `:1795`, `:1799`, `:2168-2169`). Definition pattern: `_val = lambda a: a.get_value() if hasattr(a, "get_value") else str(a)`.

**Apply to:** Any new code that reads a `PyAttribute` (e.g. `_val(cam.name)`, `_val(action.name)`). DO NOT use `_val` on `PyCoNode.type` inside `get_action_custom_ui_actions` — that field is a plain `str` (RESEARCH Pitfall 1).

### `flame.messages.show_in_dialog` for Flame-side error UX

**Source:** `flame/camera_match_hook.py:2134-2138` and dozens of other call sites.

**Apply to:** `_export_camera_from_action_selection`. Use `title="Export Camera to Blender"` to keep the error-dialog title identical to the Action-scope handler — UI-SPEC says the menu leaf name is the same; the error dialogs follow that label.

### `repr()` for embedding user input in bridge code strings

**Source:** `tools/blender/forge_sender/transport.py:23-29` (security docstring) and `:237-240` (build_payload).

**Apply to:** `make_create_code` (new) — the Action name comes from a `bpy.props.StringProperty` (user input). Use `f"{name!r}"` (which is `f"{repr(name)}"`). Never `f"...'{name}'..."`.

### Tier-1 / Transport-Tier / Remote-Tier popup pattern

**Source:** `tools/blender/forge_sender/__init__.py:162-187` (`_popup` and `_popup_multiline`), and the existing `FORGE_OT_send_to_flame.execute` body which classifies failures across tiers.

**Apply to:** `FORGE_OT_send_to_flame_choose_action`. Specifically:
