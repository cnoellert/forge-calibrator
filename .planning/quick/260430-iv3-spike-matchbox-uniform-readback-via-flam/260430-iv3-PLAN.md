<objective>
Test the channel-API path for matchbox uniform readback that the prior 260430-ddi spike never tried. The earlier spike stopped at the matchbox PyNode wrapper layer and concluded Path B was dead because float uniforms serialize as `<Channel Name="...">` pointers in `save_node_setup` output. This spike inverts that conclusion: if uniforms ARE channels, then Flame's animation/channel API should be able to walk to them — independent of the matchbox PyNode wrapper.

Three structured probes test progressively-more-permissive Flame surfaces:
1. The matchbox node's animation/channel surface (un-animated)
2. The same surface after a uniform is right-click-Animated (channels often materialize lazily)
3. The Action expression-link target enumeration (does Flame's own link picker know the matchbox channel paths?)

A fourth probe tests a sidecar Comment-node fallback **only if** probes 1-3 all die — proving an architectural pivot is unblocked.

Purpose: Either revive Path B (matchbox VP values readable from headless Python via channels) or kill it definitively with primary-source evidence so the next planner moves to a sidecar-storage architecture without re-litigating.

Output: A single PROBE.md with verdict-per-probe, full transcripts under transcripts/, and a Recommendation Matrix that names the surviving path.
</objective>
