<callback_factory_design>
The factory:

```python
# Place ABOVE _export_camera_to_blender (around line 2628, after
# _launch_blender_on_blend's end-of-function blank line).

# Discrete log10 ladder for the right-click menu — must match
# tools/blender/bake_camera.py::_FLAME_TO_BLENDER_SCALE_LADDER. The
# default-entry hardcode stays at 100.0 (studio default per the
# 260501-em8 pivot); these are the *additional* artist-pickable stops.
_LADDER_MENU_STOPS = (0.01, 0.1, 1.0, 10.0, 100.0)


def _make_export_callback(scale, *, camera_scope=False):
    """Return a single-arg callback (selection,) that fires the
    appropriate export entry point with flame_to_blender_scale=scale
    injected. The closure-over-scale lets us register one entry per
    ladder stop in the menu list without per-stop helper functions.

    camera_scope=False -> Batch right-click on Action node
                          (calls _export_camera_to_blender)
    camera_scope=True  -> Action-schematic right-click on Camera node
                          (calls _export_camera_from_action_selection)

    The wrapped functions accept a flame_to_blender_scale keyword and
    forward it to _export_camera_pipeline."""

    def _cb(selection):
        if camera_scope:
            return _export_camera_from_action_selection(
                selection, flame_to_blender_scale=scale)
        return _export_camera_to_blender(
            selection, flame_to_blender_scale=scale)

    return _cb
```

Wrapping behaviour:
- `_export_camera_to_blender(selection)` → `_export_camera_to_blender(selection, *, flame_to_blender_scale=100.0)`
- `_export_camera_from_action_selection(selection)` → `_export_camera_from_action_selection(selection, *, flame_to_blender_scale=100.0)`
- `_export_camera_pipeline(action, cam, label)` → `_export_camera_pipeline(action, cam, label, *, flame_to_blender_scale=100.0)`

Default `100.0` on each signature is the load-bearing back-compat anchor:
- The existing default menu entries (registered as
  `"execute": _export_camera_to_blender` and
  `"execute": _export_camera_from_action_selection`, with no factory
  wrapper) keep working — the menu callback is still a plain reference
  to the function, Flame calls it as `fn(selection)`, and the default
  kicks in.
- The hardcoded `flame_to_blender_scale=100.0` literal at line 2906 is
  replaced with the parameter — exact same wire value, just plumbed.
</callback_factory_design>

| Flame menu dispatch -> Python callback | Flame caches the registered `execute` callable at hook-registration time; the closure-captured `scale` value rides along. Untrusted-input risk: nil (internal tooling, hardcoded ladder values). |
| Python callback -> _export_camera_pipeline | Keyword-only `flame_to_blender_scale` flows in at the JSON layer. Out-of-ladder values would pass through this seam unchecked, but bake_camera.py's `_validate_flame_to_blender_scale` (per 260501-dpa) catches them. |
